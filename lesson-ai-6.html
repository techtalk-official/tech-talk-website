Computer Vision — How AI Sees

⏱ 20 min read 💻 Hands-on Code 🎯 Beginner

Every time you unlock your phone with your face, search Google Images, or watch a self-driving car navigate traffic — that's Computer Vision at work. It's the branch of AI that gives machines the ability to "see" and understand images and video. In this lesson, we'll understand exactly how it works, and build a real image classifier that can distinguish between different objects — using code you can run right now.

How Do Computers "See"?

Here's the fundamental question: what does a computer actually see when you show it a photo? The answer is surprisingly simple — numbers. Every image is just a grid of pixels, and every pixel is a number (or three numbers for color — Red, Green, Blue values from 0-255).

A 100×100 color photo = 100 × 100 × 3 = 30,000 numbers. That's the raw input an AI receives. From those 30,000 numbers, it must figure out whether the image contains a cat, a car, a human face, or a tumor on an X-ray.

The pixel grid

A grayscale pixel = 1 number (0=black, 255=white). A color pixel = 3 numbers (Red, Green, Blue). A standard Instagram photo at 1080×1080 pixels = 1080 × 1080 × 3 = 3,499,200 numbers that the AI processes to understand what's in the image.

CNNs — The Architecture That Changed Everything

In Lesson 5, we built a basic neural network. For images, there's a special type of neural network that works dramatically better: the Convolutional Neural Network (CNN). CNNs were the breakthrough that made modern computer vision possible.

Instead of connecting every pixel to every neuron (which would be computationally insane for large images), CNNs use a smarter approach — they scan the image with small filters, looking for specific patterns in local regions.

How CNN Layers Work:

  • Convolutional layers — Scan the image with small filters (e.g., 3×3 pixels) looking for edges, curves, textures
  • Pooling layers — Compress the detected features, making the network more efficient and robust to small shifts
  • Fully connected layers — Take the detected features and combine them into a final prediction
How CNNs learn to see

Early convolutional layers detect simple features: horizontal edges, vertical edges, corners, color gradients. Middle layers combine these into shapes: circles, rectangles, curves. Deep layers recognize complex objects: eyes, wheels, faces. This hierarchical feature detection — from pixels to objects — is why CNNs work so well for visual tasks.

Real-World Applications of Computer Vision

  • Face unlock — Your phone's camera maps 30,000+ points on your face
  • Medical imaging — AI detects tumors in X-rays and MRIs with radiologist-level accuracy
  • Self-driving cars — Real-time detection of pedestrians, signs, lane markings, other vehicles
  • Quality control — Factories use CV to detect defects on assembly lines at speeds humans can't match
  • Instagram filters — Real-time face detection and segmentation to apply effects
  • Google Lens — Point your camera at anything and get instant information

Build an Image Classifier — Step by Step

Let's build a CNN that classifies images into 10 categories (airplane, car, bird, cat, deer, dog, frog, horse, ship, truck) using the CIFAR-10 dataset — 60,000 real color images.

# Install: pip install tensorflow matplotlib numpy
import tensorflow as tf
from tensorflow import keras
import numpy as np
import matplotlib.pyplot as plt

# Class names for CIFAR-10
class_names = ['Airplane', 'Car', 'Bird', 'Cat', 'Deer',
               'Dog', 'Frog', 'Horse', 'Ship', 'Truck']

# Load dataset (downloads automatically ~170MB)
(X_train, y_train), (X_test, y_test) = keras.datasets.cifar10.load_data()

# Normalize pixel values to 0-1
X_train = X_train / 255.0
X_test = X_test / 255.0

print(f"Training images: {X_train.shape}")  # (50000, 32, 32, 3)
print(f"Test images: {X_test.shape}")        # (10000, 32, 32, 3)

# Preview some images
plt.figure(figsize=(12, 4))
for i in range(12):
    plt.subplot(2, 6, i+1)
    plt.imshow(X_train[i])
    plt.title(class_names[y_train[i][0]], fontsize=8)
    plt.axis('off')
plt.suptitle('Sample CIFAR-10 Images', fontsize=12)
plt.tight_layout()
plt.show()
# Build the CNN
model = keras.Sequential([
    # First Convolutional Block
    keras.layers.Conv2D(32, (3,3), activation='relu', padding='same',
                        input_shape=(32, 32, 3)),
    keras.layers.Conv2D(32, (3,3), activation='relu', padding='same'),
    keras.layers.MaxPooling2D(2, 2),
    keras.layers.Dropout(0.25),

    # Second Convolutional Block
    keras.layers.Conv2D(64, (3,3), activation='relu', padding='same'),
    keras.layers.Conv2D(64, (3,3), activation='relu', padding='same'),
    keras.layers.MaxPooling2D(2, 2),
    keras.layers.Dropout(0.25),

    # Classifier
    keras.layers.Flatten(),
    keras.layers.Dense(512, activation='relu'),
    keras.layers.Dropout(0.5),
    keras.layers.Dense(10, activation='softmax')  # 10 categories
])

model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

model.summary()
# Train the model
history = model.fit(
    X_train, y_train,
    epochs=10,
    batch_size=64,
    validation_split=0.1,
    verbose=1
)

# Evaluate
test_loss, test_accuracy = model.evaluate(X_test, y_test, verbose=0)
print(f"\n✅ Test Accuracy: {test_accuracy * 100:.2f}%")
# Make predictions and visualize
predictions = model.predict(X_test[:12])

plt.figure(figsize=(12, 6))
for i in range(12):
    plt.subplot(2, 6, i+1)
    plt.imshow(X_test[i])
    predicted = class_names[np.argmax(predictions[i])]
    actual = class_names[y_test[i][0]]
    confidence = np.max(predictions[i]) * 100
    color = 'green' if predicted == actual else 'red'
    plt.title(f"P: {predicted}\nA: {actual}\n{confidence:.0f}%",
              fontsize=7, color=color)
    plt.axis('off')
plt.suptitle('CNN Predictions (Green=Correct, Red=Wrong)', fontsize=11)
plt.tight_layout()
plt.show()
What to expect

After 10 epochs, your CNN should achieve roughly 75-80% accuracy on CIFAR-10. That means it correctly identifies the object in 3 out of every 4 images — using nothing but pixels and learned weights. State-of-the-art models achieve 99%+ on this dataset.

Transfer Learning — The Real-World Shortcut

In real projects, nobody trains CNNs from scratch (it takes days on expensive GPUs). Instead, they use transfer learning — taking a model already trained on millions of images and fine-tuning it for your specific task. This is how you build production-quality image classifiers in hours, not days.

# Transfer Learning with MobileNetV2 (pre-trained on ImageNet)
base_model = keras.applications.MobileNetV2(
    input_shape=(224, 224, 3),
    include_top=False,  # Remove the original classifier
    weights='imagenet'  # Use pre-trained weights
)
base_model.trainable = False  # Freeze pre-trained layers

# Add your own classifier on top
model_tl = keras.Sequential([
    base_model,
    keras.layers.GlobalAveragePooling2D(),
    keras.layers.Dense(128, activation='relu'),
    keras.layers.Dense(10, activation='softmax')
])

print("Transfer learning model built! Fine-tune on your own data.")

Challenge: Classify Your Own Images

After training your CIFAR-10 model, try classifying your own photos:

from PIL import Image
import numpy as np

# Load and preprocess your own image
img = Image.open('your_image.jpg').resize((32, 32))
img_array = np.array(img) / 255.0
img_array = img_array.reshape(1, 32, 32, 3)

# Predict
pred = model.predict(img_array)
predicted_class = class_names[np.argmax(pred)]
confidence = np.max(pred) * 100
print(f"Prediction: {predicted_class} ({confidence:.1f}% confident)")

Try photos of cars, planes, or cats — see how well your trained model generalizes to real photos!

Key Takeaway

Computers see images as grids of numbers. CNNs learn to detect patterns in those numbers — from edges to shapes to objects — through hierarchical feature learning. You've built a real CNN that classifies 10 types of objects with 75-80% accuracy. In production, transfer learning lets you achieve even higher accuracy in a fraction of the training time.
