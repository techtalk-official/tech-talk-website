Large Language Models (LLMs)

⏱ 25 min read 💻 Hands-on Code 🎯 Beginner

ChatGPT crossed 100 million users in just 2 months — faster than any technology in history. Gemini, Claude, Llama, Mistral — Large Language Models are reshaping every industry. But what actually makes them "large"? How are they trained? Why do they sometimes confidently say wrong things? And most importantly — how can you use them effectively? This lesson answers all of that, plus gives you hands-on experience calling real LLM APIs.

What Makes an LLM "Large"?

The word "large" refers to the number of parameters — the adjustable weights in the neural network. More parameters = more capacity to learn patterns from data.

Scale matters — but isn't everything

GPT-4 is estimated to have ~1 trillion parameters, but Meta's Llama models with 70 billion parameters often perform comparably on many tasks. Efficient training, better data, and fine-tuning techniques matter as much as raw size.

How LLMs Are Trained — 3 Stages

Stage 1: Pre-training (The Foundation)

The model is trained on an enormous corpus of text — essentially a large portion of the internet, books, scientific papers, and code. The task is simple: predict the next token. Given "The capital of France is ___", predict "Paris." Repeat this billions of times across trillions of tokens, and the model develops a deep statistical understanding of language, facts, and reasoning patterns.

Cost: Pre-training GPT-4 cost an estimated $100 million+ in compute. This is why only a handful of companies can do it.

Stage 2: Supervised Fine-tuning (SFT)

The raw pre-trained model is good at predicting text but not at following instructions. Human trainers create thousands of example conversations — question + ideal answer — and fine-tune the model to follow instructions and be helpful. This turns a "text predictor" into an "assistant."

Stage 3: RLHF — Reinforcement Learning from Human Feedback

Human raters compare different model responses and rank them. A "reward model" learns what humans prefer. The LLM is then fine-tuned using reinforcement learning to produce responses that score higher on the reward model. This is what makes ChatGPT feel natural and helpful — and what reduces harmful outputs.

The secret behind ChatGPT's personality

RLHF is why ChatGPT feels different from just "autocomplete on steroids." The human feedback training shaped it to be helpful, harmless, and honest — not just statistically likely. It's also why different models feel different — their RLHF data reflects different preferences and values.

Why LLMs Hallucinate

LLMs sometimes state wrong facts with total confidence — this is called hallucination. Understanding why it happens helps you use LLMs more effectively.

LLMs don't "look up" facts. They generate the most statistically plausible next token based on patterns in training data. If the training data contained many confident-sounding statements about a topic (even wrong ones), the model will confidently reproduce that pattern. It has no mechanism to "check" whether something is true — it predicts what a true-sounding answer looks like.

⚠️ Always verify critical information

Never trust LLM outputs for medical advice, legal information, financial decisions, or factual claims without verification. Use LLMs for drafting, brainstorming, explaining concepts, and writing code — not as a source of ground truth.

Call a Real LLM API — Hands On

Let's call the Gemini API (free tier available) and build three practical LLM applications.

# Install: pip install google-generativeai
import google.generativeai as genai

# Get your free API key from: https://aistudio.google.com
API_KEY = "your_gemini_api_key_here"
genai.configure(api_key=API_KEY)

# Initialize the model
model = genai.GenerativeModel('gemini-2.0-flash')

App 1 — Smart Code Explainer

def explain_code(code_snippet):
    prompt = f"""You are a coding teacher. Explain this code clearly:
    
```python
{code_snippet}
```

Explain:
1. What this code does (in simple terms)
2. How it works step by step
3. A real-world use case for it
Keep it beginner-friendly."""

    response = model.generate_content(prompt)
    return response.text

# Test it
sample_code = """
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

print([fibonacci(i) for i in range(10)])
"""

explanation = explain_code(sample_code)
print("📚 Code Explanation:")
print(explanation)

App 2 — Tech Content Generator

def generate_instagram_post(topic, style="educational"):
    prompt = f"""Create an Instagram post for a tech channel about: {topic}
    
Style: {style}
Requirements:
- Start with a hook that grabs attention
- Explain the concept clearly in 3-4 lines
- Include 1 surprising fact
- End with an engaging question for comments
- Add 10 relevant hashtags
- Use emojis naturally
- Language: English with some Hindi/Marathi words for Indian audience

Format the output clearly with sections."""

    response = model.generate_content(prompt)
    return response.text

# Generate posts for different topics
topics = ["What is Prompt Engineering?", "Python vs JavaScript in 2025"]
for topic in topics:
    print(f"\n📱 Post for: {topic}")
    print("=" * 50)
    print(generate_instagram_post(topic))
    print()

App 3 — Interactive Chat with Memory

def run_tech_chatbot():
    """A simple chatbot that remembers conversation history"""
    chat = model.start_chat(history=[])
    
    # Set the persona
    system_message = """You are TechBot, an AI assistant for Tech Talk Official channel.
    You help Indian students and developers learn technology concepts.
    Keep answers concise, practical, and beginner-friendly.
    Use examples relevant to Indian developers."""
    
    print("🤖 TechBot: Namaste! I'm TechBot from Tech Talk Official.")
    print("Ask me anything about AI, Web Dev, Python, or tech careers!")
    print("Type 'quit' to exit\n")
    
    while True:
        user_input = input("You: ").strip()
        
        if user_input.lower() in ['quit', 'exit', 'bye']:
            print("🤖 TechBot: Thank you! Keep learning! 🚀")
            break
            
        if not user_input:
            continue
        
        # Include system context with first message
        full_message = f"{system_message}\n\nUser question: {user_input}" if len(chat.history) == 0 else user_input
        
        response = chat.send_message(full_message)
        print(f"\n🤖 TechBot: {response.text}\n")

# Run the chatbot
run_tech_chatbot()
What you just built

Three production-ready LLM applications: a code explainer, an Instagram content generator, and a contextual chatbot with memory. These are real-world use cases companies pay developers thousands of rupees to build. You just built all three in under 100 lines of code.

Prompt Engineering — Getting Better Results

The quality of your LLM output depends heavily on how you write your prompt. Here are the key techniques:

  • Be specific — "Explain recursion with a Python example for a 16-year-old" beats "Explain recursion"
  • Give a role — "You are a senior software engineer at Google..." sets context
  • Specify format — "Return a JSON object with keys: title, description, hashtags"
  • Show examples — Few-shot prompting: "Here's an example of what I want: [example]. Now do the same for..."
  • Chain of thought — "Think step by step before answering" dramatically improves reasoning
  • Set constraints — "In exactly 3 bullet points" or "In under 100 words"

Key Takeaway

LLMs are trained in 3 stages: pre-training on massive text data, supervised fine-tuning on instruction-following examples, and RLHF to align with human preferences. They generate text by predicting the most likely next token — which is why they can hallucinate. You've now called a real LLM API and built three practical applications. Prompt engineering is the skill that separates average LLM users from power users.
